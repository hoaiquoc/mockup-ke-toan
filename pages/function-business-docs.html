Gợi ý mapping nhanh để chạy ổn định và dễ refactor

Kiến trúc mapping nên dùng

  • Lưu giao dịch bằng account_id trong journal lines.
  • Tạo lớp dictionary map account_id -> { code, name, type, status }.
  • Mọi báo cáo chỉ aggregate theo account_id, hiển thị code/name ở lớp view.
  • Cho phép đổi code/name mà không phá dữ liệu lịch sử.

Contract ổn định cho API

  • Response trả cả accountId và object accountView.
  • accountView gồm code, name, type để FE không phải join thêm.
  • Thêm scope=branch|all cho mọi API report.
  • Scope all phải tổng hợp theo account_id rồi mới render theo code.

Checklist triển khai nhanh

  1. Tạo cache dictionary account theo version hoặc updatedAt.
  2. Read-through cache: miss thì query DB và cập nhật cache.
  3. Đặt cờ cảnh báo khi journal chứa account_id không còn active.
  4. Viết 1 adapter duy nhất map dữ liệu report -> view model.
  5. Test refactor: đổi code tài khoản, kiểm tra số liệu lịch sử không đổi.
